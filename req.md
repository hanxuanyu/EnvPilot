# EnvPilot 需求文档（初稿）

## 1. 文档信息
- 项目名称：EnvPilot
- 技术栈：Go + React + shadcn/ui + Tailwind CSS + Wails
- 文档版本：v0.1（初稿）
- 日期：2026-03-06
- 状态：待评审

## 2. 背景与目标
### 2.1 背景
在多环境（开发、测试、预发、生产等）运维场景中，服务器与中间件（MySQL、Redis、MQ）数量增长快，信息分散在文档、脚本、命令行历史和人员经验中，导致以下问题：
- 查询与操作入口不统一，切换成本高。
- 高风险操作缺乏统一审计。
- 环境内对象多、命名混乱，定位慢。
- 多套网络下地址不统一，连接方式复杂。

### 2.2 目标
构建一个桌面端管理工具，统一管理多环境下的服务器和中间件，并提供高频运维能力：
- 快速执行服务器命令/脚本。
- 快速查询 MySQL/Redis。
- 快速发送 MQ 消息。
- 为服务配置域名并通过内置 DNS 服务完成解析。
- 按环境分组组织资产，提高可视化和检索效率。

### 2.3 非目标（当前阶段）
- 不替代企业级堡垒机。
- 不提供完整 CI/CD 流水线能力。
- 不作为公网权威 DNS 服务。

## 3. 用户角色与使用场景
### 3.1 角色
- 管理员：最高权限，负责环境、资产、凭据、域名、DNS、审计与系统配置管理。
- 普通只读用户：无需登录即可查看所有环境配置和运行状态，不具备新增、修改、删除、执行操作权限。

### 3.2 典型场景
- 场景 A：在“测试环境”批量执行日志采集脚本。
- 场景 B：在“预发环境”快速查询某业务库中的订单记录。
- 场景 C：向指定 MQ Topic/Queue 发送测试消息并查看回执。
- 场景 D：通过内置 DNS 域名快速访问服务，无需记忆 IP/端口。

## 4. 业务对象与术语
- 环境（Environment）：一组隔离的资源集合，如 dev/test/staging/prod。
- 分组（Group）：环境内的逻辑分类，如“核心服务”“数据层”“缓存层”。
- 资产（Asset）：可管理对象，分为服务器与中间件实例。
- 服务域名（Service Domain）：为资产或服务绑定的 DNS 名称。
- 连接配置（Connection Profile）：协议、地址、端口、认证信息。
- 操作记录（Operation Log）：命令、查询、消息发送等行为审计记录。

## 5. 功能需求
### 5.1 环境与分组管理
1. 支持创建/编辑/删除环境。
2. 支持环境排序与启用/禁用。
3. 每个环境下支持多级或单级分组（建议先单级，降低复杂度）。
4. 支持将服务器和中间件归类到分组。
5. 支持跨分组检索（按名称、标签、类型、域名）。

### 5.2 服务器管理
1. 支持维护服务器基础信息：
- 名称、IP、端口、协议（SSH/RDP/WinRM 等）、系统类型、标签、备注。
2. 支持维护认证方式：
- 用户名+密码、密钥、跳板机（可选）。
3. 服务器凭据字段要求：
- 用户名（必填）
- 密码（可选，按协议）
- 私钥（可选）
- 私钥口令（可选）
- 凭据说明与标签（可选）
4. 所有凭据必须加密存储，页面默认脱敏展示，管理员可按需临时明文查看（需二次确认）。
5. 支持连接测试与状态检测（在线/离线/未知）。
6. 支持批量导入/导出服务器信息（CSV/JSON，后续可扩展）。
7. 支持轻量服务端状态查询：
- CPU、内存、磁盘、负载、网络连通性（Ping/TCP）
- 进程存活检测（按进程名或端口）
- 最近一次状态更新时间

### 5.3 中间件管理
#### 5.3.1 MySQL
1. 维护连接信息：Host、Port、DB、用户、认证方式。
2. 连接测试、库表列表浏览。
3. SQL 执行（只读优先），支持限制语句类型。
4. 查询结果表格展示、导出（CSV）。

#### 5.3.2 Redis
1. 维护连接信息：Host、Port、DB、认证、TLS（可选）。
2. 支持常用命令查询（GET/HGETALL/SCAN 等），并可配置命令白名单。
3. 结果结构化展示。

#### 5.3.3 MQ（通用抽象）
1. 支持维护 MQ 连接（先定义统一模型，首批落地 RocketMQ、RabbitMQ、SofaMQ）。
2. 支持发送消息：Topic/Queue、Key、Headers、Body。
3. 提供消息模板与历史复用。
4. 可选回执/发送结果展示。

### 5.4 快速操作中心
1. 在指定服务器执行单条命令。
2. 在指定服务器执行脚本文件（Bash/PowerShell）。
3. 支持单机执行与批量执行。
4. 支持预设脚本模板，参数化输入。
5. 执行结果实时输出、状态标记（成功/失败/超时）。
6. 高风险命令二次确认（如 `rm -rf`、`DROP` 关键字策略）。
7. 集成在线终端能力：
- 支持 Web Terminal（Wails 内嵌）交互式会话。
- 仅支持 Linux 服务器（SSH 协议）。
- 支持终端会话连接、断开、重连与超时控制。
- 支持终端操作审计（命令输入与执行结果摘要）。
8. 支持“快捷操作面板”：
- 常用命令收藏
- 按环境复用命令模板
- 一键复制执行结果

### 5.5 域名与 DNS 服务
1. 可为每个服务或资产维护一个或多个域名。
2. 工具内置 DNS 服务，支持 A 记录（优先），可扩展 CNAME/SRV。
3. 域名解析目标可指向：
- 服务器 IP
- 中间件 Host
- 逻辑服务地址（映射到具体资产）
4. 支持按环境隔离解析视图，避免不同环境同名冲突。
5. 支持 TTL 配置与本地缓存。
6. 支持 DNS 日志查看（请求来源、域名、命中结果、时间）。
7. 支持在 UI 中一键复制“可访问域名 + 端口”。

### 5.6 检索与视图
1. 全局搜索：按名称/IP/域名/标签/分组。
2. 环境视图 + 分组树 + 列表联动。
3. 收藏与最近访问对象。

### 5.7 审计与日志
1. 记录以下行为：
- 登录/退出
- 命令执行
- SQL 执行
- Redis 命令执行
- MQ 消息发送
- 配置变更
2. 支持按用户/时间/环境/对象类型筛选。
3. 支持导出审计日志。

### 5.8 配置备份与恢复
1. 支持整体配置导出（加密包）。
2. 支持从备份恢复。
3. 支持自动备份策略（可选）。

### 5.9 运行状态看板（轻量）
1. 支持按环境展示服务器与中间件健康状态总览。
2. 支持最近检查时间、可用率趋势（近 24 小时，轻量统计）。
3. 支持异常对象高亮与快速跳转到对应资产详情页。
4. 支持轻量轮询健康检查（可配置轮询间隔）并保留手动触发（避免引入复杂监控系统）。

### 5.10 系统配置管理（YAML）
1. 系统配置统一使用 YAML 文件管理（如：系统参数、DNS 端口、轮询间隔、只读访客开关等）。
2. 提供管理员可视化配置界面，支持表单化编辑 YAML 配置项。
3. 支持配置保存前校验（必填、类型、范围、枚举值）。
4. 支持配置版本快照与回滚（轻量级）。
5. 敏感配置项（密钥、密码）在界面中默认脱敏并受权限控制。

## 6. 非功能需求
### 6.1 安全性
- 凭据加密存储（系统密钥链或主密码派生密钥）。
- 敏感字段脱敏展示。
- 支持操作权限控制（管理员/普通只读用户）。
- 高风险操作强提醒和审计留痕。
- 普通只读用户免登录访问由开关控制；开启后仅允许查看，所有写操作和执行操作仅管理员可用。

### 6.2 性能
- 资产规模目标：单环境 500+ 资产可流畅检索。
- 命令执行输出实时显示，首屏响应 < 1 秒（本地可达场景）。
- DNS 查询响应目标：本地网络下 p95 < 50ms。

### 6.3 可用性
- Windows 优先，兼容 macOS/Linux（后续评估）。
- 断网情况下可进行本地配置管理。
- 异常恢复：重启后恢复上次会话上下文（可选）。

### 6.4 可维护性
- 模块化架构（资产、执行、查询、DNS、审计分层）。
- 插件化协议适配（MQ 类型、认证方式可扩展）。

## 7. 技术与架构约束（建议）
### 7.1 客户端架构
- 前端：React + TypeScript + shadcn/ui + Tailwind CSS。
- 桌面容器：Wails。
- 后端服务层：Go。

### 7.2 数据存储（建议）
- 本地元数据：SQLite。
- 敏感凭据：加密后落库。
- 操作日志：本地文件或 SQLite 分表。
- 系统配置：YAML 文件（支持可视化读写与版本快照）。

### 7.3 模块划分（建议）
- `asset`：环境/分组/资产管理。
- `executor`：命令与脚本执行。
- `terminal`：在线终端会话管理。
- `connector`：MySQL/Redis/MQ 连接抽象。
- `dns`：DNS 解析与缓存。
- `health`：资产健康检查与状态聚合。
- `config`：YAML 配置读写、校验与回滚。
- `audit`：审计日志。
- `auth`：登录与权限。

## 8. 数据模型（草案）
### 8.1 Environment
- id, name, code, description, enabled, sort_order, created_at, updated_at

### 8.2 Group
- id, environment_id, name, type, sort_order, created_at, updated_at

### 8.3 Asset
- id, environment_id, group_id, asset_type(server/mysql/redis/mq), name, host, port, tags, status, remark

### 8.4 Credential
- id, asset_id, auth_type, username, encrypted_password, encrypted_private_key, encrypted_passphrase, key_ref, updated_at

### 8.5 ServiceDomain
- id, environment_id, asset_id, domain, record_type, target, ttl, enabled

### 8.6 OperationLog
- id, user_id, op_type, environment_id, asset_id, command_or_sql_or_payload, result, duration_ms, created_at

### 8.7 HealthSnapshot
- id, environment_id, asset_id, cpu_usage, memory_usage, disk_usage, load_avg, ping_ms, tcp_status, process_status, checked_at

### 8.8 SystemConfig
- id, config_key, config_value_yaml, version, updated_by, updated_at

## 9. 交互与页面（MVP）
1. 管理员登录页（普通只读用户免登录）。
2. 环境总览页（环境卡片 + 资产统计）。
3. 资产管理页（服务器/MySQL/Redis/MQ Tab）。
4. 快速操作页（命令执行、脚本执行、在线终端、历史记录）。
5. 数据查询页（MySQL/Redis）。
6. MQ 发送页（模板 + 发送日志）。
7. DNS 管理页（域名映射 + 查询日志）。
8. 运行状态页（健康看板 + 异常对象）。
9. 系统配置页（管理员可视化编辑 YAML 配置）。
10. 审计页。

## 10. 里程碑建议
### M1（MVP）
- 环境/分组/资产 CRUD
- 服务器命令执行（单机）
- 在线终端（Linux SSH，单会话）
- MySQL 查询（只读）
- Redis 常用查询
- MQ 发送（RocketMQ、RabbitMQ、SofaMQ）
- 轻量轮询健康检查与状态看板
- 域名映射 + DNS A 记录解析
- 审计基础能力
- 管理员登录 + 普通只读免登录浏览（开关控制）
- YAML 配置管理 + 管理员可视化配置页

### M2（增强）
- 批量执行与脚本模板
- 高风险策略引擎
- 备份恢复
- DNS 高级记录与冲突检测

### M3（扩展）
- 多协议插件生态
- 团队协作能力（多用户、多端同步）
- 远程代理与复杂网络拓扑支持

## 11. 验收标准（MVP）
1. 可创建至少 3 个环境并分组管理 100+ 资产。
2. 可对目标服务器执行命令并获取结果。
3. 可通过在线终端与目标服务器进行交互式会话。
4. 可执行 MySQL 只读 SQL 并展示结果。
5. 可执行 Redis 常用命令查询。
6. 可向 RocketMQ、RabbitMQ、SofaMQ 成功发送消息。
7. 域名可被内置 DNS 成功解析到目标。
8. 可在状态看板查看资产健康状态并识别异常对象。
9. 管理员可执行写操作，普通只读用户在开关开启时可免登录查看但不可修改。
10. 所有关键操作可在审计页面查询。
11. 管理员可通过可视化界面修改 YAML 系统配置并生效。

## 12. 风险
- 不同 MQ 协议差异大，统一抽象可能牺牲高级特性。
- 内置 DNS 与本机网络配置可能存在权限或端口占用问题。
- 高风险操作防护规则需要持续调优。
- YAML 手工修改可能引入格式错误，需完善校验与回滚机制。

## 13. 附录：建议优先级（MoSCoW）
- Must：环境分组、资产管理、凭据加密、命令执行、Linux 在线终端、MySQL/Redis 查询、RocketMQ/RabbitMQ/SofaMQ 发送、DNS 映射解析、轻量轮询健康看板、审计日志、管理员/只读双角色、YAML 配置与可视化管理。
- Should：批量执行、脚本模板、高风险策略、备份恢复。
- Could：插件化扩展、跨端同步、复杂网络拓扑支持。
- Won't（当前阶段）：替代堡垒机、完整发布流水线。
